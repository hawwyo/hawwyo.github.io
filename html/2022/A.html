Problem A. Amazing Trick
Time limit: 3 seconds
Memory limit: 1024 megabytes
Alice is a magician and she creates a new trick. She has ncards with dierent numbers from 1to nwritten
on them. First, she asks an audience member to shue the deck and put cards in a row. Lets say the
i-th card from the left has the number aion it.
Then Alice picks two permutations pand q. There is a restriction on pand qpermutations cant
have xed points. Which means i:pi6=i and qi6=i.
After permutations are chosen, Alice shues the cards according to them. Now the i-th card from the
left is the card a[p[q[i]]. The trick is considered successful if i-th card from the left has the number ion
it after the shues.
Help Alice pick the permutations pand qor say it is not possible for the specic starting permutation a.
Input
The rst line of the input contains the number of tests t(1t105).
Each test is described in two lines. The rst line contains one integer n the number of cards
(1n105). The second line contains nintegers ai the initial permutation of the cards (1ain;
i6=j:ai6=aj).
It is guaranteed that the sum of nover all tests does not exceed 105.
Output
Print the answer for each test case in the same order the cases appear in the input.
For each test case, print Impossible in a single line, if no solution exists.
Otherwise, print Possible in the rst line, and in the following two lines print permutations pand q.
